# System Security Plan (SSP)

The SSP documents your security posture and control implementations.

## Generating Your SSP

Navigate to your package and click **Generate SSP** or access the **SSP** tab.

![Screenshot: SSP generation page](/img/screenshots/cmmc/ssp-generate.png)

## SSP Readiness Requirements

Before generating, ensure you have:

- [ ] System Name completed
- [ ] System Owner assigned
- [ ] Security Officer (ISSO) assigned
- [ ] At least 50% of controls assessed

The readiness checklist shows your current status.

![Screenshot: SSP readiness checklist](/img/screenshots/cmmc/ssp-readiness.png)

## SSP Contents

The auto-generated SSP includes:

| Section | Description |
|---------|-------------|
| **System Identification** | Basic system information |
| **System Description** | Purpose, boundaries, and architecture |
| **Roles & Responsibilities** | Key personnel and their roles |
| **Control Statements** | Implementation details for each requirement |
| **Asset Inventory Summary** | Systems and components in scope |
| **POA&M Summary** | Current remediation status |

## Downloading the SSP

After generation:

- **View Full SSP** - Review in browser
- **Download PDF** - Save for submission

![Screenshot: SSP download options](/img/screenshots/cmmc/ssp-download.png)
